# Alchat 内容安全监测系统 - Windows 启动说明

## 项目简介

Alchat 内容安全监测系统是一个基于 Next.js 16 + React 19 + TypeScript 构建的内容安全监测平台，提供以下核心功能：

- **对话监测**：实时对话内容安全检测，包含风险评分与敏感词匹配
- **检测日志**：完整的检测记录查询，支持按风险等级、关键词筛选
- **统计仪表盘**：可视化数据展示，包含趋势图、分布图、热词排行
- **策略配置**：灵活的阈值与处置动作配置，支持多策略管理
- **敏感词库**：分类分级敏感词管理，支持四级分类（严重/高危/中危/低危）

## 安装步骤

### 1. 安装 Node.js

1. 访问 [Node.js 官网](https://nodejs.org/) 下载 LTS 版本（推荐 v20.x）
2. 运行安装程序，**勾选 "Add to PATH"**
3. 安装完成后，打开 **命令提示符** 或 **PowerShell**，验证安装：

```bash
node --version
# 应输出 v20.x.x

npm --version
# 应输出 10.x.x
```

### 2. 安装 pnpm

```bash
npm install -g pnpm
```

验证：

```bash
pnpm --version
# 应输出 9.x.x
```

> 如果遇到权限问题，请以管理员身份运行命令提示符

### 3. 解压项目

将项目压缩包 `alchat-content-safety.zip` 解压到你想要的目录，例如：

```
D:\projects\alchat-content-safety
```

### 4. 安装依赖

打开终端，进入项目目录：

```bash
cd D:\projects\alchat-content-safety
```

安装项目依赖：

```bash
pnpm install
```

> 首次安装可能需要几分钟，请耐心等待。如果遇到网络问题，可配置国内镜像：
> ```bash
> pnpm config set registry https://registry.npmmirror.com
> ```

## 启动项目

### 开发模式（推荐）

```bash

```

启动成功后，终端会显示：


打开浏览器访问 **http://localhost:5000** 即可使用系统。

```bash

# 再启动
pnpm start
```

同样访问 **http://localhost:5000**。

## 功能使用指南

### 对话监测

1. 在首页输入框中输入消息
2. 点击发送或按回车，系统自动执行内容安全检测
3. 右侧面板实时展示检测结果：
   - 风险分数（0-100）
   - 风险等级（安全/低/中/高/严重）
   - 处置动作（放行/警告/拦截）
   - 分数拆解（基础分 × 乘数因子 + 上下文调节）
   - 命中敏感词列表

### 检测日志

1. 点击左侧导航"检测日志"
2. 支持按风险等级过滤（安全/低/中/高/严重）
3. 支持关键词搜索
4. 显示完整检测记录，包含内容、等级、分数、处置、命中词、时间

### 统计仪表盘

1. 点击左侧导航"统计仪表盘"
2. 查看概览数据：总检测量、今日检测、通过率、拦截率
3. 近7天检测趋势柱状图
4. 处置分布、风险等级分布、命中分类分布
5. 敏感词命中排行 TOP 10

### 策略配置

1. 点击左侧导航"策略配置"
2. 查看当前激活策略及所有策略列表
3. 可编辑策略的阈值和处置动作
4. 可新建自定义策略
5. 可激活/删除策略（默认策略不可删除）

#### 阈值说明

| 等级 | 默认阈值 | 含义 |
|------|----------|------|
| 低危 | 20分 | 20分以上为低风险 |
| 中危 | 45分 | 45分以上为中风险 |
| 高危 | 70分 | 70分以上为高风险 |
| 严重 | 90分 | 90分以上为严重风险 |

#### 处置动作说明

| 动作 | 含义 |
|------|------|
| 放行 | 内容安全，正常通过 |
| 警告 | 内容发出，但提示风险 |
| 拦截 | 内容被阻止，不发送 |
| 人工审核 | 内容需人工审核后决定 |

### 敏感词库

1. 点击左侧导航"敏感词库"
2. 查看所有敏感词及其等级、分类、分值
3. 支持按等级和分类过滤
4. 可添加新敏感词（输入词、选择分类、等级、分值）
5. 可启用/禁用/删除敏感词

#### 等级分类说明

| 等级 | 标签 | 典型分类 | 分值范围 |
|------|------|----------|----------|
| 1级 | 严重 | 恐怖主义、政治敏感 | 90-100 |
| 2级 | 高危 | 违法信息、色情暴力、危害生命 | 70-90 |
| 3级 | 中危 | 歧视言论、宗教敏感、隐私侵犯 | 40-65 |
| 4级 | 低危 | 垃圾信息、不良信息 | 10-40 |

## 风险评分算法说明

风险评分采用三级计算模型：

```
最终分 = min(基础分 × 乘数因子 + 上下文调节, 100)
```

1. **基础分**：各匹配敏感词分值的加权求和
   - 1级词权重 ×1.5
   - 2级词权重 ×1.2
   - 3级词权重 ×1.0
   - 4级词权重 ×0.7

2. **乘数因子**：多词叠加惩罚
   - `1 + 0.3 × (匹配词数 - 1)`
   - 匹配词越多，惩罚越重

3. **上下文调节**：基于匹配密度和文本长度
   - 高密度（>10%）：+15分
   - 中密度（>5%）：+5分
   - 长文本稀释（>500字）：-5分

## 常见问题

### Q: 启动时端口被占用怎么办？

修改 `.coze` 文件中的端口号，或在启动时指定端口：

```bash
pnpm dev -- -p 3000
```

### Q: pnpm install 报错怎么办？

1. 清除缓存：`pnpm store prune`
2. 删除 `node_modules` 和 `.next` 目录后重试
3. 检查 Node.js 版本是否 >= 18.17
4. 配置国内镜像源：`pnpm config set registry https://registry.npmmirror.com`

### Q: 页面样式异常？

1. 清除浏览器缓存
2. 删除 `.next` 目录后重新启动
3. 确保使用现代浏览器（Chrome 90+、Edge 90+、Firefox 90+）

### Q: 如何修改默认端口？

项目默认运行在 5000 端口。如需修改：

1. 修改 `.coze` 文件中的 `run` 命令端口号
2. 修改 `package.json` 中 `dev` 和 `start` 脚本的端口参数
